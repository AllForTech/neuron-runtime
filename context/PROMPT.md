======================================================================
GLOBAL DESIGN SYSTEM (STRICT)
======================================================================
- THEME: Deep Obsidian (#050505) with Zinc (#71717a) and pure White (#FFFFFF) accents.
- CORNERS: Ultra-smooth rounded corners (24px to 32px radii) for all containers and cards.
- BORDERS: Subtle 1px borders using a semi-transparent Zinc gradient (#27272a).
- TYPOGRAPHY: Inter or Geist Sans. Headlines: Bold, tight tracking. Body: Light-weighted, increased line-height.
- ANIMATION: Liquid-smooth transitions (Framer Motion style). Avoid jitter. Focus on "orbital" motion and soft "breathing" glows.
- LAYOUT: Spacious, high-margin, centered composition to match the premium "SaaS-meets-Infrastructure" feel.

======================================================================

======================================================================
PURPOSE: Immediate technical positioning and high-impact visual identity.
CONTENT GOAL: "Beyond Code." Establish Neuron as an autonomous orchestration kernel.

VISUAL COMPONENT:
- Central "Orbital Core": A high-fidelity, monochrome 3D sphere or particle cluster representing the Runtime.
- Node Constellation: Floating, glowing icons for APIs, Databases, and AI models orbiting the core with thin, pulsing connection lines.
- Navigation: Minimal top bar (Features, Capabilities, Demo, Docs, Pricing) with a sleek "Start Building" button.

COPY:
- HEADLINE: BEYOND CODE.
- SUB-HEADLINE: Autonomous backend orchestration kernel. Build visually, deploy instantly.
- CTAs: Primary [Get Started] (White bg), Secondary [Explore Platform] (Ghost button with arrow icon).

STRICTNESS: Must mirror the top half of neuron-engine-landing-page.png exactly in hierarchy and spacing.

======================================================================
SECTION 2: SYSTEM DETERMINISM (CORE PILLARS)
======================================================================
PURPOSE: Break down the four non-negotiable engineering principles of the engine.
CONTENT GOAL: Transition from "Visuals" to "Value."

VISUAL COMPONENT:
- 4-Column Bento Grid: Four equal-width cards with 24px rounded corners.
- Iconography: Minimalist line-art icons centered in small Zinc circles.
- Hover State: A soft, white radial glow that follows the cursor within the card.

THE FOUR PILLARS:
1. DETERMINISTIC EXECUTION: "Every event. Every time. Deterministic by design."
2. RESILIENT BY DESIGN: "Retries, compensation, and fault tolerance built-in."
3. FULL OBSERVABILITY: "Trace every node. Visualize every decision."
4. LIMITLESS EXTENSIBILITY: "Integrate anything. Build without boundaries."

STRICTNESS: No fluff. Each card must contain exactly one title, one icon, and one sentence of supporting text.

======================================================================
SECTION 3: THE EXECUTION PIPELINE (ORCHESTRATION FLOW)
======================================================================
PURPOSE: Explain the procedural logic of the engine without technical jargon overload.
CONTENT GOAL: Connect Systems -> Orchestrate Logic -> Execute Actions.

VISUAL COMPONENT:
- Split Layout: Left side features a smaller version of the "Connectivity Orb." Right side features a vertical, stepped execution log.
- Connectivity Orb: Surround the orb with service logos (AWS, Kafka, Slack, Stripe, GitHub, PostgreSQL, OpenAI).
- Stepped Log (Execution Pulse):
    - 01 Event Ingestion: "Capture events from APIs, queues, webhooks, and databases."
    - 02 Workflow Execution: "Apply business logic and AI decisions to determine the path."
    - 03 Action & Delivery: "Execute across systems with full traceability."

STRICTNESS: Use a thin vertical line to connect the three steps, showing a "pulse" moving through them as the user scrolls.
